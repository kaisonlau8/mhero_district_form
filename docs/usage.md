# 使用文档

## 1. 文档目标

本文档面向日常使用该工具的业务同事，重点说明：

- 这个工具能做什么
- 需要准备哪些文件
- 如何在网页模式或 macOS 桌面模式中使用
- 导出结果长什么样
- 常见问题如何处理

如果你要修改代码、调整规则或重新打包，请看：

- [开发文档](development.md)
- [打包与发版文档](release.md)

## 2. 工具能做什么

该工具用于自动处理以下 7 份日报：

1. `续保投保率`
2. `新保投保率`
3. `首保实施率`
4. `去年同期交付未新保车辆`
5. `门店备件库存导出`
6. `二保实施率`
7. `保养提醒任务列表`

系统会把这些源报表的数据自动写入“区域各指标情况一览xxxx”模板中的对应工作表，然后导出新的结果 Excel。

## 3. 支持的运行方式

### 3.1 网页模式

适合：

- 开发环境
- 本机调试
- 不需要桌面安装包的场景

启动方式：

```bash
cd /Users/i/myCode/报表项目
./start.command
```

打开：

- `http://127.0.0.1:8000`

### 3.2 macOS 桌面模式

适合：

- 直接给使用者安装
- 不想手动打开浏览器
- 希望像普通应用一样双击打开

桌面版会把网页界面封装在应用窗口中，并通过系统保存对话框导出 Excel。

## 4. 需要准备的文件

### 4.1 必传文件

必须提供以下 7 份 `.xlsx` 文件：

| 业务文件名关键字 | 说明 |
| --- | --- |
| `门店备件库存导出` | 写入模板中的 `备件库存明细` |
| `保养提醒任务列表` | 写入模板中的 `招揽实施率` |
| `首保实施率` | 写入模板中的 `首保` |
| `二保实施率` | 写入模板中的 `二保` |
| `新保投保率` | 写入模板中的 `新保` |
| `续保投保率` | 与“去年同期交付未新保车辆”一起写入 `续保` |
| `去年同期交付未新保车辆` | 与“续保投保率”一起写入 `续保` |

### 4.2 可选模板

系统默认使用内置模板：

- `app/assets/report_template.xlsx`

如果当天模板临时有变化，也可以在页面中额外上传一个新的“区域各指标情况一览xxxx”文件作为模板。

## 5. 文件识别规则

系统不是靠完整文件名匹配，而是靠“文件名前缀/关键字”识别。

例如：

- `续保投保率20260319171212.xlsx`
- `续保投保率_导出_0327.xlsx`
- `续保投保率(1).xlsx`

只要文件名中还能识别出 `续保投保率` 这个关键字，系统就会把它识别为正确的源文件。

这意味着每天导出文件后面带不同时间戳、后缀或括号编号，通常都不影响使用。

## 6. 数据写入对应关系

系统固定按以下关系回填模板：

| 模板工作表 | 来源文件 |
| --- | --- |
| `备件库存明细` | `门店备件库存导出` |
| `招揽实施率` | `保养提醒任务列表` |
| `首保` | `首保实施率` |
| `二保` | `二保实施率` |
| `新保` | `新保投保率` |
| `续保` | `续保投保率` + `去年同期交付未新保车辆` |

## 7. 内置业务规则

### 7.1 去年同期交付未新保车辆

该报表的列结构与 `续保投保率` 不完全一致。

系统会在其 `E` 列与 `F` 列之间自动补 1 列空白列，再追加写入 `续保` 工作表。

### 7.2 门店备件库存导出

该报表的 `M` 列有时会以文本形式导出。

系统会自动把 `M` 列转成数字格式，以确保模板中的公式可以正常参与计算。

### 7.3 首保 H 列自动重算

`首保` 工作表中的 `H` 列表示“是否首保”。

由于 DMS 在“超期首保”和“二次交付”场景下，源报表里的 `H` 列可能显示为“否”，但 `I` 列其实已经存在“实际首保日期”，因此系统会按以下规则重算：

- 如果 `I` 列有数据，则 `H` 列强制写为 `是`
- 如果 `I` 列没有数据，则 `H` 列强制写为 `否`

也就是说，导出结果中的 `首保` 工作表，`H` 列将以 `I` 列为准，而不是完全照搬源报表。

## 8. 导出文件规则

### 8.1 导出文件名

导出文件名固定格式为：

- `区域各指标情况一览MMDD.xlsx`

例如 3 月 27 日导出时，文件名会是：

- `区域各指标情况一览0327.xlsx`

### 8.2 公式计算

模板中的 `总表` 依赖其他工作表公式统计。

系统会保留模板公式并将工作簿设置为“打开时自动重算”。因此：

- 文件导出后请使用 Excel 打开
- 打开后 `总表` 会根据模板公式自动计算

## 9. 网页模式使用步骤

### 9.1 启动服务

```bash
cd /Users/i/myCode/报表项目
./start.command
```

### 9.2 打开页面

浏览器访问：

- `http://127.0.0.1:8000`

### 9.3 上传文件

在页面中：

1. 把 7 份日报拖入上传区域
2. 或点击“选择文件”后批量选择这 7 个 Excel
3. 如需替换模板，再点击“替换模板”上传新的模板文件

### 9.4 检查识别状态

页面会显示 7 个状态卡片：

- 已识别的文件会显示文件名
- 未识别或缺失的文件会显示“未上传”

只有 7 份源报表全部识别成功后，“生成并下载 Excel”按钮才会可用。

### 9.5 生成文件

点击：

- `生成并下载 Excel`

系统处理成功后，浏览器会开始下载结果文件。

## 10. macOS 桌面版使用步骤

### 10.1 打开应用

双击：

- `区域报表自动生成.app`

如果首次打开提示来源未知：

1. 在 Finder 中找到应用
2. 右键应用
3. 选择“打开”
4. 再确认一次

### 10.2 上传文件

桌面版页面与网页模式一致：

1. 拖入 7 份源报表
2. 如有需要，再上传模板
3. 等待 7 个状态卡片全部变为已识别

### 10.3 保存结果文件

点击“生成并下载 Excel”后：

- 桌面版不会走浏览器下载
- 会弹出 macOS 系统保存对话框
- 选择保存位置并确认后，系统会把 Excel 写到本地

如果未手动输入 `.xlsx` 后缀，系统也会自动补齐。

## 11. 常见问题

### 11.1 按钮一直是灰色，不能点击

原因通常是 7 份源报表没有全部识别成功。

建议检查：

- 是否缺少某一份报表
- 文件是否是 `.xlsx`
- 文件名里是否还保留了关键字，如 `续保投保率`、`新保投保率`

### 11.2 页面提示“有 N 个文件未识别，已忽略”

说明你拖进来了额外文件，或者文件名里不包含系统能识别的关键字。

系统会忽略这些文件，不影响其他已识别文件。

### 11.3 模板没有用上我刚上传的那个文件

如果页面里额外上传了模板文件，系统会优先使用你上传的模板。

如果没有上传模板，则使用内置模板：

- `app/assets/report_template.xlsx`

### 11.4 导出后总表没有立刻显示最终值

这是因为总表依赖 Excel 自身的公式重算。

处理方式：

- 用 Microsoft Excel 打开导出文件
- 等待其自动刷新计算

### 11.5 桌面版导出时报错

先检查：

- 是否已选择具体文件名而不是文件夹
- 保存位置是否有写入权限
- 是否被系统安全策略拦截

如果是第一次使用桌面版，请尽量先通过 Finder 的“右键 -> 打开”完成首次信任。

### 11.6 打开应用时提示“已损坏”或“无法验证开发者”

当前桌面包是未签名版本，这在企业内部分发时比较常见。

处理方式通常有两种：

- Finder 中右键应用，选择“打开”
- 在系统“隐私与安全性”里允许打开

## 12. 使用建议

- 每次导出前，先确认 7 份源报表都是当天最新版本
- 如果模板更新过，建议同步替换内置模板，避免每次都手工上传
- 如果发现某天导出文件名规则变化，优先确认文件关键字是否仍保留
- 如果未来新增新的业务报表或工作表，请联系维护人员同步改规则，不建议直接手工篡改导出的结果文件
